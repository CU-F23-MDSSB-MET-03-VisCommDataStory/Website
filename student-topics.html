2 Topics

  1. Visualization and comparison of distributions of numerical variables (Boxplots, Violin Plots, Histograms, Density plots, Ridgeline plots) (Inspiration from the stories https://www.data-to-viz.com/story/OneNum.html, https://www.data-to-viz.com/story/OneNumOneCatSeveralObs.html, and https://www.data-to-viz.com/caveat/boxplot.html)
  2. Visualization of one categorical variable or nested categorical variables (Barplots and Lollipops and versions of stacking and dodging, Pie charts, Treemaps, Word Clouds) (Inspiration from the stories https://www.data-to-viz.com/story/OneNumOneCat.html, https://www.data-to-viz.com/story/SevCatOneNumNestedOneObsPerGroup.html, https://www.data-to-viz.com/caveat/multi_distribution.html, and https://www.data-to-viz.com/caveat/pie.html)
  3. Visualization of two or three numerical variables (Scatter plots, Scatter plots with summary statistics - regression or loess, the overplotting problem, density and contour plots, What can be used for the thrid variable – size, color, …?) (Inspiration from the stories https://www.data-to-viz.com/story/TwoNum.html and https://www.data-to-viz.com/story/ThreeNum.html)
  4. Two numerical variables where one is ordered (e.g. time) and an additional categorical one (Line plots, Area plots, Focusing, the Spaghetti plot problem) (Inspiration from the stories https://www.data-to-viz.com/story/TwoNumOrdered.html, https://www.data-to-viz.com/story/OneCatSevOrderedNum.html, and https://www.data-to-viz.com/caveat/spaghetti.html)

Optional topics:

  1. Dendro grams (Hierarchical Clustering)
  2. Networks (Inspiration from https://www.data-to-viz.com/story/AdjacencyMatrix.html)
  3. Maps (Inspiration from https://www.data-to-viz.com/graph/choropleth.html, https://www.data-to-viz.com/story/GPSCoordWithoutValue.html, and https://www.data-to-viz.com/story/GPSCoordWithValue.html)
  4. Interactive graphs (plotly, shiny, or observable)
  5. Tableau and Power BI

Generally, for all specific visualization there is more information on https://datavizcatalogue.com and https://www.data-to-viz.com.

3 Structure and Format

Structure of the student presentation:

  • Explain the underlying visualization task
  • Provide examples of solutions using different data sources
  • Compare different solutions with the same data
  • Finish with a summary about good and bad uses for the visualization type, common problems, do’s and don’ts

Format:

  • Provide a “hands-on” document where code and data is provided which produce visualizations. It can be a markdown or notebook file in which you mix text and code or a raw script where you put some descriptive text in comments. It can be in R or python. (If you want to use something else please coordinate before.) A repository in the organization https://github.com/JU-F22-MDSSB-MET-03 can be provided for distribution to others. Contact me on Teams. Students will then be enabled to clone it and play with the code.
  • You do not need presentation slides but you can. You can use revealjs in quarto to nicely present code and visualizations.
